# 开发计划

## 阶段一：Rust CLI MVP（当前阶段）

**周期**: 2-4 周

**目标**: 完成最小可用 CLI 工具。

**已实现功能**:
- `dumbcoder init` — 初始化项目配置
- `dumbcoder ask "问题"` — 代码库问答
- `dumbcoder explain <文件>` — 解释文件/函数
- `dumbcoder search "查询"` — 智能代码搜索
- 调用本地 Ollama 模型服务
- 基础安全过滤（文件黑名单、路径沙箱）
- TOML 配置文件支持

**验收标准**:
- [x] 可以在开发机运行
- [x] 可以连接本地 Ollama 模型服务
- [x] 可以对真实仓库进行代码问答
- [x] 可以解释指定文件或函数
- [x] 回答能返回相关文件路径

## 阶段二：代码索引与 AST

**周期**: 4-6 周

**目标**: 提升代码定位和上下文准确率。

**计划功能**:
- `dumbcoder index` — 构建代码索引
- 文件索引
- 符号索引
- tree-sitter 函数级切分
- AST 上下文提取
- Git changed files 增量索引
- 本地索引数据库 (sqlite-vec)

**验收标准**:
- 能识别函数、类、方法
- 能按 symbol 查询代码
- 能定位相关函数代码块
- 检索结果明显优于纯 grep

## 阶段三：TUI 交互界面

**周期**: 3-5 周

**目标**: 提升终端交互体验。

**计划功能**:
- `dumbcoder tui` — 交互式 TUI
- 对话区
- 文件引用区
- 代码片段区
- diff 预览区
- 测试结果区
- 快捷键操作
- 历史任务浏览

**验收标准**:
- 开发人员可以在 TUI 中完成代码问答
- 可以查看 AI 引用的文件和代码片段
- 可以浏览 diff
- 可以确认或拒绝 patch

## 阶段四：测试生成与 Review

**周期**: 4-6 周

**目标**: 进入真实开发工作流。

**计划功能**:
- `dumbcoder test` — 单元测试生成
- `dumbcoder review` — Git diff 审查
- 单元测试生成
- git diff review
- 测试命令识别
- 测试执行和日志摘要
- Review 报告生成

## 阶段五：受控 Patch Agent

**周期**: 6-8 周

**目标**: 支持小范围代码修改。

**计划功能**:
- `dumbcoder patch` — 生成受控代码修改
- 修改计划生成
- diff 生成
- `git apply --check`
- 用户确认应用
- 自动运行测试
- 失败回滚
- 审计日志
